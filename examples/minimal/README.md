# Minimal Example

## Overview

Demonstrates the most basic Storytime usage with a simple component hierarchy.

## Structure

- One Site with a single Section ("Components")
- One Subject ("Heading") under the Section
- One Story with props only, no custom title or description
- Simple Heading component that renders a name prop

## Key Features

- Single component (Heading) with minimal props (name)
- Props inheritance from Subject target
- No optional fields used (no custom Story title/description)
- Basic Site → Section → Subject → Story hierarchy
